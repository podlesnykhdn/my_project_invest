## Ключевые правила разработки

### Перед любым изменением
1. Читать актуальный файл через API — не работать по памяти
2. После изменения JS — проверять синтаксис через `node --check`
3. Не использовать `const lp=` дважды в одной области видимости
4. При пересборке index.html — всегда из `/tmp/dashboard_template.html`

### Дашборд (index.html)
- Один HTML файл, весь JS внутри `<script>` без внешних библиотек
- Данные встроены в `_E` (const _E = {...}) — embedded fallback
- `lmap` строится из `_E.portfolio.positions` (НЕ из лога!) — гарантия цены TGLD
- TGLD цена — из Tinkoff API (primary), MOEX TQTF ненадёжен
- Перед загрузкой на GitHub — JS проверяется через `node --check`
- PORTFOLIO JS массив должен совпадать с rules.json

### collector.py
- Запускается в GitHub Actions — MOEX ISS может блокировать
- Tinkoff API — основной источник для ETF и реального портфеля
- `sync_portfolio_from_tinkoff()` — обновляет qty в rules.json автоматически
- `QUAL_ONLY_TICKERS` — фильтр акций только для квалов, проверяется через Tinkoff
- `check_and_filter_anomalies()` — проверяет новые тикеры, ежемесячная ревизия

### rules.json (v3.3)
Структура: `portfolio.positions`, `fundamentals` (EPS/P/E),
`debt_data` (Долг/EBITDA), `tinkoff_names`, `dividends_config`,
`qual_only_tickers`, `_meta`

## Как работать с этим проектом

1. **Изменения в collector.py** → проверить синтаксис `ast.parse()`
2. **Изменения в index.html** → проверить `node --check` перед загрузкой
3. **Новые данные в rules.json** → обновить embedded `_E` в index.html
4. **Qty изменились** → обновить в rules.json, index.html (PORTFOLIO JS), advisor_bot.py
5. **Новый квал-only тикер** → добавить в QUAL_ONLY_TICKERS в collector.py И advisor_bot.py
6. **Пересборка дашборда** → использовать `/tmp/dashboard_template.html` как основу
